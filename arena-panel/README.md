# Beckon Arena Panel — head-to-head AI judging + ELO

The machinery for the **Arena Score**: cross-vendor AI judges compare two models' gauntlet artifacts blind, majority vote picks a winner, winners climb an ELO ladder. This is the head-to-head layer ARENA.md Phase 2 describes — implemented.

**Ruling #2 boundary (read first):** this tool produces the Arena Score ONLY. It never reads or writes `score.json`, never suggests Human Scores, and the two boards are published separately. The disagreements between them are content, not a bug.

## How a match works

1. Pick a test (01–05) and two models that both have a `results/<model>/<test>/output/` artifact.
2. The artifacts are anonymized (vendor/model names redacted from source, capped at 60k chars) and presented as **Model A / Model B**.
3. A panel of **three cross-vendor judges** is seated from `config.json`. **No self-judging:** any judge sharing a vendor with either contestant is skipped (vendor read from `results/<model>/meta.json → provider` — fill it in!).
4. Each judge sees the artifacts in **independently randomized order** (position bias is real), plus screenshots when the judge has vision (and the reference image for test 05).
5. Each judge must vote A or B — **no ties possible with three votes**. A judge that times out or returns garbage twice is replaced by the next alternate on the bench (the "panel stuck on standby" lesson). Fewer than three seated votes = match void.
6. The match record (per-judge votes + reasoning + cost + latency) is written to `matches/`. Nothing is hidden — per-judge votes get published.

ELO (start 1000, K=24) is **recomputed from the match log every time** — the log is the source of truth, the ladder is derived. `leaderboard` writes `elo.json` for the site to read.

## Usage

```bash
export OPENROUTER_API_KEY=sk-or-...   # not needed for --dry-run

node arena-panel/arena.mjs check                                     # verify judge model ids exist on OpenRouter
node arena-panel/arena.mjs match 02-lava-lamp claude-fable-5-max gpt-5-6-sol-ultra
node arena-panel/arena.mjs season                                    # every missing pairing, all judgeable tests
node arena-panel/arena.mjs leaderboard                               # recompute + print ladder, write elo.json
```

`--dry-run` exercises the whole pipeline with deterministic mock votes (no API, no cost) — use it to test wiring. Dry matches are marked and excluded from the real ladder. `--verbose` prints judge raw output and debug detail. `--rematch` replays already-played pairings.

## Cost controls

`season` stops when cumulative spend hits `budget_usd_per_run` (default $20; OpenRouter returns per-call cost when usage reporting is on). Every match record carries `total_cost_usd`.

## Known limits (documented, not hidden)

- **Test 04 is weakly blinded** — a self-portrait can reveal its author no matter how much source text we redact (flagged `weak_blinding` in config). Publish the caveat with the results.
- **Tests 06 (Remotion) and 08 (Blender) are not arena-judgeable yet** — their artifacts are videos/renders; judging them needs multimodal video review. Future work.
- **Test 07 is excluded on purpose** — it's scored against a secret answer key; a judge without the key can only reward confident-sounding reports, which is exactly the failure mode the bench exists to catch.
- Judge `model_id`s in `config.json` are best-effort — run `check` before the first paid run and fix any that moved.

## Files

```
config.json    judges (with vendor + vision flags), judgeable tests, ELO params, budget, redaction list
arena.mjs      the whole harness (no dependencies, Node 22+)
matches/       one JSON per match — append-only source of truth
elo.json       derived ladder, regenerated by `leaderboard` — the site reads this
```
